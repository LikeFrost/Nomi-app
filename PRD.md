# Nomi — Product Requirements Document

## 1. 产品概述

**Nomi** 是一款交互式虚拟伴侣应用，用户通过点击互动来陪伴一个名为 Nomi 的 3D 小女孩动画角色。角色拥有情绪与精力两个维度的状态，由 AI（Claude Haiku）驱动真实的情绪反应，随时间自然衰减，形成持续性的陪伴体验。

### 目标用户

- 希望在碎片化时间获得陪伴感的用户
- 对虚拟伴侣 / 养成类应用有兴趣的用户
- 年龄层：18–35 岁，偏好精致、有温度的视觉风格

### 平台

iOS、Android、Web（Expo 跨平台）

## 2. 核心功能需求

### 2.1 互动系统

| 动作 | Emoji | 情绪影响 | 精力影响 |
|------|-------|----------|----------|
| 打招呼（Greet） | 👋 | +0.18 | −0.08 |
| 亲亲（Kiss）   | 💋 | +0.25 | −0.06 |
| 加油（Cheer）  | 🎉 | +0.18 | −0.20 |

- 单次点击触发动画，500ms 节流防止重复触发
- 新动作可打断正在播放的动画（预抢占过渡）

### 2.2 情绪模型

- 双轴状态：`mood`（心情）、`energy`（精力），取值范围 `[0, 1]`
- **饱和增量**：接近上限时增益减缓，接近下限时消耗减缓，避免极端值
- **自然衰减**：精力约 3 分钟衰减周期，心情约 5 分钟；心情衰减受精力状态制约（精力低时心情恢复更慢）
- 状态持久化至本地存储，后台时长自动计算衰减量

### 2.3 AI 驱动行为

- 调用 Claude Haiku 模型，根据最近 8 条行为历史、点击频率和当前情绪状态输出：
  - `emotion`：happy / tired / playful
  - `pace`：动画播放速度系数
  - `intensity`：动画强度
  - `boredom`：无聊程度（影响是否跳过动画）
- **即发即忘（fire-and-forget）**：立即使用本地降级逻辑，后台异步更新 AI 结果，下次触发时生效，确保零延迟
- 3 次失败后进入 30 秒冷却，防止过度重试
- 系统提示词启用 Prompt Caching，降低 Token 成本

### 2.4 3D 角色渲染

- 使用 Three.js + expo-gl 在移动端提供 WebGL 渲染
- 角色模型：Nomi 小女孩（girl-opt.glb），包含四段动画剪辑：Idle、Greeting、Kiss、Cheering
- 动画间平滑过渡（crossfade），跳过每段剪辑前 ~80ms 的姿态稳定帧，保证响应即时
- ACES Filmic 色调映射 + 多光源布光方案（半球光、环境光、主光、补光）

### 2.5 每日计数器

- 记录当天与角色互动的次数（"今日已给 N 次反应"）
- 每日零点自动重置，前台恢复时校验日期

### 2.6 彩蛋机制：状态栏渐显

- 默认隐藏心情/精力状态栏
- 当日累计互动超过 4 次后，状态栏在第 4–7 次点击间渐入显示
- 提供探索发现感，避免初次打开信息过载

### 2.7 加载动画

- 启动时展示带脉冲光环和爱心图标的 Splash 动画
- 角色模型加载完成后自动淡出

## 3. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | AI 调用使用本地降级，点击响应 < 50ms |
| 稳定性 | 精力/心情衰减使用递归 setTimeout（非 setInterval），防止后台积压 |
| 离线可用 | 核心互动和动画在无网络时可用（AI 功能降级至本地逻辑） |
| 本地化 | 界面文案为中文 |
| 适配 | 支持 iOS、Android、Web 三端 |

---

## 4. 视觉设计规范

- 风格：软糖 Bento 风格，圆角、柔和阴影、无硬边框
- 主色：`#FDF2F8`（浅粉背景）、`#F472B6`（热粉强调色）
- CTA 色：`#22C55E`（绿色）
- 心情条：粉色；精力条：紫色（`#A78BFA`）
- 圆角 Token：20px（默认）、12px（小）、999px（胶囊）

---

## 5. 技术约束

- LLM 接入：通过自定义代理（`EXPO_PUBLIC_LLM_BASE_URL`）调用，Web 端绕过 CORS
- 模型：`claude-haiku-4-5-20251001`，超时 1.5 秒
- 本地存储：AsyncStorage 持久化情绪状态和每日计数
- 构建：Expo EAS

---
